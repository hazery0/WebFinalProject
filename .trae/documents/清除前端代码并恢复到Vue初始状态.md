# 清除前端代码并恢复到Vue初始状态

## 一、项目分析
- 当前前端使用Vue 3 + TypeScript + Vite
- 需要恢复到初始空状态，同时确保能连接到后端
- 后端运行在 `http://localhost:8080/api`

## 二、实施步骤

### 1. 清理现有代码
- 删除 `src/` 目录下除核心文件外的所有内容
- 保留 `main.ts`、`App.vue` 和基本目录结构
- 移除 `stores/`、`services/` 等非必要目录

### 2. 重建基础结构
创建最小化的Vue项目文件：
- `main.ts`：初始化Vue应用和路由
- `App.vue`：简单的模板结构，包含 `router-view`
- `router/index.ts`：配置基础路由（仅首页）
- `views/HomeView.vue`：简单的首页组件

### 3. 添加必要依赖
- 确保 `axios` 已安装（用于API请求）
- 确保 `vue-router` 配置正确

### 4. 配置后端连接
- 创建简单的API服务，配置基础URL为 `http://localhost:8080/api`
- 实现基本的请求拦截器（可选）
- 确保与后端CORS配置兼容

### 5. 测试连接
- 编写简单的测试代码，验证前端能成功调用后端API
- 测试登录或注册接口，确保数据能正常传递

### 6. 验证功能
- 确保前端能正常启动
- 确保与后端的连接正常
- 确保基础页面能正常渲染

## 三、技术要点
- 保持代码简洁，只保留必要功能
- 确保后端连接配置正确
- 测试前后端通信是否正常
- 确保项目能正常构建和运行

## 四、预期结果
- 前端恢复到Vue初始空状态
- 能成功连接到后端API
- 能发送和接收数据
- 项目结构清晰，便于后续扩展

## 五、风险控制
- 备份原有代码（可选，防止误操作）
- 逐步清理，确保核心功能不受影响
- 测试每一步的修改，确保正常运行

## 六、执行时间
- 预计完成时间：30-45分钟

## 七、验证标准
- 前端能正常启动并显示首页
- 能成功调用后端API并获取响应
- 项目能正常构建（`npm run build`）

通过以上步骤，将前端恢复到初始状态，并确保与后端的连接正常，为后续开发奠定基础。
